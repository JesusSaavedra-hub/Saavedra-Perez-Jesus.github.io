Caracteristicas Pagina web estatica Pagina web dinamica
Formato Para crear una página web estática de forma directa y funcional, el formato estándar consiste en un archivo HTML que estructura el contenido, un archivo CSS para el diseño y una carpeta de activos para imágenes. El flujo de trabajo más eficiente actualmente es redactar el contenido en Markdown y utilizar un Generador de Sitios Estáticos (SSG) para compilarlo en archivos finales listos para ser servidos por un servidor web sin necesidad de una base de datos activa. Para que una página sea dinámica el usuario envía una solicitud al servidor y este ejecuta un script en lenguajes como PHP o Python que consulta una base de datos para extraer información específica que luego inserta automáticamente en una plantilla HTML prediseñada creando así un archivo único en tiempo real que se envía de vuelta al navegador para que el usuario vea su contenido personalizado junto con código JavaScript que permite actualizar elementos visuales sin necesidad de recargar toda la ventana.
Diseño El diseño de una página web estática se centra en la claridad y la velocidad, ya que al no tener que procesar datos en el servidor, puede permitirse una estética más refinada y una carga instantánea. Los estilos modernos suelen seguir una estructura de "una sola página" (Single Page Application) o aterrizaje (Landing Page) con secciones bien definidas. . el diseño de una página web dinámica se basa en una maqueta flexible dividida en contenedores donde el código frontend hecho con HTML y CSS define los espacios fijos mientras que el backend inyecta el contenido variable permitiendo que elementos como galerías de imágenes o listas de productos se adapten automáticamente al tamaño de la pantalla mediante frameworks como React o Vue que gestionan los cambios de estado sin que el usuario note saltos en la interfaz logrando una experiencia fluida y personalizada según los datos que lleguen desde el servidor
Estructura La estructura se resume en: un archivo index.html (contenido), una carpeta CSS (diseño) y otra de imágenes. Visualmente, el orden es: Header (logo/menú), Hero (título/botón), Cuerpo (servicios/texto) y Footer (contacto). Todo se sirve directo del servidor al navegador sin procesos intermedios. La estructura de una página dinámica se organiza en capas donde el header y el footer suelen ser componentes estáticos reutilizados en todo el sitio mientras que el body funciona como un contenedor vacío o placeholder que se rellena automáticamente con datos extraídos de una base de datos mediante etiquetas de programación permitiendo que una misma rejilla o layout muestre contenido distinto según el usuario o la categoría seleccionada separando siempre la presentación visual en archivos CSS de la lógica de datos para que el diseño sea escalable y se mantenga consistente sin importar cuánta información nueva se agregue al servidor
